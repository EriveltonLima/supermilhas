# Segurança

## Relatar uma vulnerabilidade

Envie detalhes para `erivelton.lima.tech@gmail.com` antes de publicar informações que possam colocar usuários em risco.

Não inclua senhas, tokens, números de fidelidade, dados bancários ou páginas completas contendo informações pessoais.

## Modelo de segurança

- não há backend ou banco de dados;
- não há autenticação própria;
- não há execução de código remoto;
- configurações ficam em `chrome.storage.local`;
- links externos usam HTTPS e são abertos somente por ação do usuário;
- a chave Pix é pública e utilizada exclusivamente para doação voluntária.
